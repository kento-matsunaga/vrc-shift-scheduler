# Quick Start - テスター向けガイド

VRC Shift Scheduler のテスト版へようこそ！

## 重要な注意事項

- **テスト用です**：本番環境での使用は推奨しません
- **バグがあれば報告してください**

---

## アクセス情報

### テスト環境 URL

- **フロントエンド**: http://localhost:5173 (ローカル開発時)
- **API**: http://localhost:8080

### テスト用アカウント

シードデータ投入後、以下のアカウントでログインできます：

| メール | パスワード | ロール |
|--------|-----------|--------|
| admin@example.com | password123 | owner |

## はじめての操作フロー

### 1. ログイン

1. フロントエンドURLにアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック

→ ダッシュボードに自動遷移します

### 2. イベントを作成

1. 左メニューから「イベント」をクリック
2. 「新規作成」ボタンをクリック
3. イベント情報を入力：
   - **イベント名**: `テストイベント`
   - **種別**: `通常営業` または `特別営業`
   - **説明**: `テスト用イベント`
4. 「作成」ボタンをクリック

→ イベント一覧に新しいイベントが表示されます

### 3. 営業日を追加

1. イベント一覧から作成したイベントの「営業日」をクリック
2. 「営業日を追加」ボタンをクリック
3. 営業日情報を入力：
   - **日付**: 今日以降の日付
   - **開始時刻**: `21:00`
   - **終了時刻**: `23:30`
4. 「追加」ボタンをクリック

→ 営業日がカレンダーに表示されます

### 4. シフト枠を作成

1. 営業日一覧から作成した営業日をクリック
2. 「シフト枠を追加」ボタンをクリック
3. シフト枠情報を入力：
   - **ポジション**: ドロップダウンから選択
   - **枠名**: `受付A`
   - **開始時刻**: `21:00`
   - **終了時刻**: `22:00`
   - **必要人数**: `2`
4. 「追加」ボタンをクリック

→ シフト枠が一覧に表示されます

### 5. メンバーを登録

1. 左メニューから「メンバー」をクリック
2. 「新規登録」ボタンをクリック
3. メンバー情報を入力：
   - **表示名**: `テスト太郎`
4. 「登録」ボタンをクリック

→ メンバーが一覧に表示されます

### 6. シフトを割り当てる

1. シフト枠一覧画面で「割り当て」ボタンをクリック
2. メンバー選択プルダウンから割り当てたいメンバーを選択
3. 「確定」ボタンをクリック

→ シフトが確定され、割り当て状況が更新されます

## 主な機能

### イベント管理
- イベントの作成・編集・削除
- 定期パターン設定（毎週○曜日）
- イベント種別（通常/特別）

### 営業日管理
- 営業日の追加・編集・削除
- 月次カレンダー表示
- 日付・時刻の設定

### シフト枠管理
- シフト枠の作成・編集・削除
- シフトテンプレートからの一括作成
- 割り当て状況の確認（例: 1/2 人）

### シフト割り当て
- メンバーのシフト枠への割り当て
- 満員チェック（必要人数を超える割り当ては不可）
- 割り当てのキャンセル

### メンバー管理
- メンバーの登録・編集・削除
- ロールの割り当て

### 出欠確認
- 出欠確認の作成
- 公開URLの発行
- 回答の集計

### 日程調整
- 日程調整の作成
- 候補日の設定
- 公開URLの発行
- 回答の集計

## よくある質問

### Q: ログアウトするには？

A: 画面右上のユーザーメニューから「ログアウト」をクリックしてください。

### Q: シフトが割り当てられない？

A: 以下を確認してください：
- シフト枠の必要人数に達していないか（例: 2/2 は満員）
- メンバーが正しく登録されているか
- 営業日とシフト枠が正しく作成されているか

### Q: 画面が表示されない？

A: 以下を試してください：
1. ブラウザをリロード（Ctrl+R または Cmd+R）
2. キャッシュをクリア（Ctrl+Shift+Del または Cmd+Shift+Del）
3. 別のブラウザで試す

### Q: API エラーが出る？

A: サーバーが起動しているか確認してください。
バックエンドが `http://localhost:8080` で起動している必要があります。
