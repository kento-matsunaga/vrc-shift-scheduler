# Quick Start - Public Alpha テスター向けガイド

VRC Shift Scheduler の Public Alpha テスト版へようこそ！🎉

## ⚠️ 重要な注意事項

- **これは Alpha 版です**：データが予告なく消える可能性があります
- **テスト用です**：本番環境での使用は推奨しません
- **バグがあります**：バグを見つけたら報告してください

## 📌 アクセス情報

### テスト環境 URL

- **フロントエンド**: https://your-frontend-domain.onrender.com
- **API（開発者向け）**: https://your-backend-domain.onrender.com

### テスト用アカウント

このシステムは「メンバー登録」することで使い始めることができます。  
表示名を入力するだけで、すぐに利用開始できます。

---

## 🚀 はじめての操作フロー

### 1. ログイン/識別

1. フロントエンドURLにアクセス
2. 表示名を入力（例: `テスト太郎`）
3. 「ログイン」ボタンをクリック

→ イベント一覧画面に自動遷移します

### 2. イベントを作成

1. 「新しいイベントを作成」ボタンをクリック
2. イベント情報を入力：
   - **イベント名**: `テストイベント`
   - **説明**: `Alpha版テスト用イベント`
3. 「作成」ボタンをクリック

→ 作成したイベントが一覧に表示されます

### 3. 営業日を追加

1. イベント一覧から作成したイベントをクリック
2. 「営業日を追加」ボタンをクリック
3. 営業日情報を入力：
   - **日付**: `2025-12-10`（今日以降の日付）
   - **開始時刻**: `21:00`
   - **終了時刻**: `23:30`
4. 「追加」ボタンをクリック

→ 営業日が一覧に表示されます

### 4. シフト枠を作成

1. 営業日一覧から作成した営業日をクリック
2. 「シフト枠を追加」ボタンをクリック
3. シフト枠情報を入力：
   - **役職名**: `受付`
   - **インスタンス名**: `受付1`
   - **開始時刻**: `21:00`
   - **終了時刻**: `22:00`
   - **必要人数**: `2`
   - **優先度**: `1`
4. 「追加」ボタンをクリック

→ シフト枠が一覧に表示されます

### 5. メンバーを登録（別のユーザーとして）

1. 画面右上の「ログアウト」ボタンをクリック（または別のブラウザを開く）
2. 別の表示名を入力（例: `テスト花子`）
3. 「ログイン」ボタンをクリック

→ 新しいメンバーとしてログインします

### 6. シフトを割り当てる

1. シフト枠一覧画面で「割り当て」ボタンをクリック
2. メンバー選択プルダウンから割り当てたいメンバーを選択
3. 「確定」ボタンをクリック

→ シフトが確定され、確定済みメンバー一覧に表示されます

### 7. 自分のシフトを確認

1. 画面右上のナビゲーションから「マイシフト」をクリック
2. 自分に割り当てられたシフト一覧が表示されます

---

## 🔍 主な機能

### イベント管理
- イベントの作成・一覧表示
- イベント詳細の閲覧

### 営業日管理
- 営業日の追加・一覧表示
- 日付・時刻の設定

### シフト枠管理
- シフト枠の作成・一覧表示
- 役職・時間帯・必要人数の設定
- 割り当て状況の確認（例: 1/2 人）

### シフト割り当て
- メンバーのシフト枠への割り当て
- 満員チェック（必要人数を超える割り当ては不可）
- 確定済みメンバー一覧の表示

### マイシフト
- 自分に割り当てられたシフトの一覧表示
- 日付・時刻・役職の確認

## 📊 テストデータ

初期状態では、以下のテストデータが投入されています：

- **イベント**: 1件（テストイベント）
- **営業日**: 7日分（今日から7日間）
- **メンバー**: 5人（田中太郎、佐藤花子、鈴木一郎、高橋美咲、伊藤翔太）
- **シフト枠**: 各営業日に3枠（受付、案内、配信）

これらのデータを使ってすぐにテストを開始できます。

## 💡 よくある質問

### Q: ログアウトするには？

A: 現在のバージョンでは、ブラウザのローカルストレージをクリアしてください：
1. ブラウザの開発者ツールを開く（F12）
2. Application タブ → Local Storage → サイトURL
3. `member_id` と `tenant_id` を削除
4. ページをリロード

または、別のブラウザ/シークレットモードを使用してください。

### Q: データが消えた？

A: Alpha版では定期的にデータリセットが行われる可能性があります。  
重要なデータは外部に保存しておいてください。

### Q: シフトが割り当てられない？

A: 以下を確認してください：
- シフト枠の必要人数に達していないか（例: 2/2 は満員）
- メンバーが正しく登録されているか
- 営業日とシフト枠が正しく作成されているか

### Q: 画面が表示されない？

A: 以下を試してください：
1. ブラウザをリロード（Ctrl+R または Cmd+R）
2. キャッシュをクリア（Ctrl+Shift+Del または Cmd+Shift+Del）
3. 別のブラウザで試す

### Q: API エラーが出る？

A: サーバーが再起動中の可能性があります。数分待ってから再試行してください。
