# コードレビュー

コミット前の変更に対するセキュリティと品質レビュー:

1. 変更ファイルを取得: `git diff --name-only HEAD`

2. 各変更ファイルに対してチェック:

**セキュリティ問題（Critical）:**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクション脆弱性
- 入力バリデーションの欠如
- テナント分離の欠如

**DDD/アーキテクチャ（High）:**
- レイヤー境界の侵害（UsecaseがInfraをインポート等）
- リポジトリパターンの違反
- ドメインバリデーションの欠如

**コード品質（High）:**
- 50行超の関数
- 500行超のファイル
- 4レベル超のネスト
- エラーハンドリングの欠如
- fmt.Println / log.Printf の残留

**ベストプラクティス（Medium）:**
- チケットなしのTODO/FIXME
- テストの欠如
- ドキュメントの欠如

3. レポート生成:
   - 重要度: CRITICAL, HIGH, MEDIUM, LOW
   - ファイル位置と行番号
   - 問題の説明
   - 修正案

4. CRITICALまたはHIGH問題がある場合はコミットをブロック

セキュリティ脆弱性のあるコードは絶対に承認しない！
